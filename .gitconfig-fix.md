# Fix para Git que abre Vim

## Problema
Git está configurado para usar Vim como editor, lo que causa que se abra al hacer `git pull` cuando hay conflictos.

## Solución

### Opción 1: Cambiar el editor a nano (más fácil)
En la terminal del hosting, ejecuta:
```bash
git config --global core.editor "nano"
```

### Opción 2: Usar --no-edit al hacer pull
Siempre que hagas pull, usa:
```bash
git pull --no-edit
```

### Opción 3: Configurar para que no pida mensaje de merge
```bash
git config --global pull.rebase false
git config --global merge.commit no-edit
```

## Para salir de Vim si ya está abierto
1. Presiona `Esc` para asegurarte de estar en modo comando
2. Escribe `:q!` y presiona Enter (esto sale sin guardar)
3. O escribe `:wq` y presiona Enter (esto guarda y sale)
